# Security Policy

## Supported versions

Security support scope varies by repository. Refer to each repository’s release and maintenance policy when available.

## Reporting a vulnerability

Please do not open public issues for suspected vulnerabilities.

Use GitHub Security Advisories (private vulnerability reporting) for the affected repository when enabled. If not available, contact maintainers through the organization’s designated security reporting channel.

When reporting, include:

- Affected repository and component
- Vulnerability description and impact
- Reproduction details or proof of concept
- Suggested remediation (if known)

## Response expectations

Maintainers aim to:

- Acknowledge reports promptly
- Validate and triage severity
- Provide status updates during remediation
- Coordinate disclosure after a fix is available
