# Contributing Guidelines

Thanks for contributing. This repository provides organization-wide community health standards that apply by default across repositories.

## Before you start

- Check existing issues and pull requests to avoid duplicates.
- Use the appropriate issue form (bug, feature, incident, technical debt).
- Keep discussions respectful and constructive.

## Contribution process

1. Open or reference an issue that explains the problem or proposal.
2. Keep changes focused on a single concern.
3. Submit a pull request using the PR template.
4. Respond to review feedback promptly and clearly.

## Commit and pull request expectations

- Use clear, descriptive commit messages.
- Explain the motivation, scope, and impact of changes.
- Include validation evidence where applicable.
- Update documentation when behavior or process changes.

## Issue quality standards

- Include enough context for maintainers to triage quickly.
- Provide reproducible steps for bug reports.
- For feature requests, describe the problem and expected value.
- For incidents, include a concise timeline and impact statement.

## Scope for this repository

This `.github` repository should contain shared governance artifacts only:

- Issue forms and template configuration
- Pull request templates
- Community standards and policies

Do not add application code, CI/CD workflows, or project-specific configurations here.
